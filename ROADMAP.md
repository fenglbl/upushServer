# ROADMAP

`upushServer` 后续计划按“先止血、再增强、后规范化”的顺序推进。

## P0 - 当前优先级最高

- [ ] 增加基础日志能力
  - 请求访问日志
  - 异常日志
  - 安全审计日志
- [ ] 为 `/pushMessage` 增加更明确的鉴权方案
- [ ] 完善推送链路错误回传
- [ ] 梳理运行时错误的统一返回格式

## P1 - 近期可做

- [ ] 补充部署说明
- [ ] 增加环境区分（dev / prod）
- [ ] 整理配置项说明与默认值
- [ ] 继续补充接口文档
- [ ] 评估是否补 WebSocket / 实时能力说明

## P2 - 后续优化

- [ ] 更系统的参数校验
- [ ] 更细粒度的权限控制
- [ ] 日志落盘 / 结构化日志
- [ ] 监控与告警接入
- [ ] 更规范的错误码体系

## 备注

当前仓库刚从主仓库拆分出来，优先目标不是“做重型重构”，而是：

1. 保持可运行
2. 保持可部署
3. 提高安全性与可维护性
4. 逐步补齐可观测性
